煮熟的Contributions飞了？

发表于 2016-04-22   |  

问题背景

最近突然发现我在Github上的Blog提交都没被计算Contributions，以下是解决过程，特此记录

这篇主要讲一下我遇到的问题和如何解决：一直在github上写blog，最近发现我每次提交，竟然没有被算contributions，也就是没有出现绿方块，这让我有点郁闷啊，虽说写blog不是会了哪几个绿色方块，但是该有的还是得有啊，没有绿色方块也不方便看自己更新的情况啊。。。。。

怎么样才会被算Contributions?

关于这个Github官方的解释在这里
Why are my contributions not showing up on my profile?

不过呢，是英文的，23333，勉强能看懂，简单翻译一下

Issues and pull requests

  • 一年之内执行的

  • Fork的仓库不算，要独立的仓库

Commits

  • 一年之内Commit的操作

  • Commit的Email的地址与Github关联

  • 在一个独立仓库提交，Fork的不行

  • 提交的分支：

    • 默认分支（一般是master）

    • gh-pages分支（包含Project Pages sites的仓库）

另外，至少满足以下条件中的一个（主要针对对不是你创建的仓库的Commit）：

  • 仓库的协作者，或者是仓库的组织中的一员

  • Fork的仓库

  • 对这个仓库发起过Pull Request或者Issue

  • Star这个仓库

Contributions没被计算的常见原因

  • 进行Commits的Email没有被关联到Github帐号。

  • 没在仓库的默认分支进行Commit。

  • 不是对独立仓库Commit而是一个Fork仓库

  • 提交合并和压合的Pull Request

找回我煮熟的Contributions

定位原因

根据Github官方的解释，我很快找到我的Contributions飞走的原因，那就是Commit的Email地址不对，我的Email为XXX@XX.Local，这样的Email自然Github不认识了。

对症下药

首先为了以后的Commits能被正确计算，我先改掉我提交的Eamil地址。

  • 可以用Github Desktop的GUI界面更改，在偏好设置里

  • 可以命令行直接改

那么以前的Contributions怎么办？绝不能就这样让他们飞了啊，想要找回之前的Contributions，就要改掉之前提交记录的Email。

Github官方也有解决方案Changing author info ，嗯，同样是英文的。。。

使用脚本更改仓库的Git历史

  • 打开Terminal

  • 对你的仓库创建一个新的Clone，并前往目录

  • 复制粘贴以下脚本，根据你的信息修改以下变量
    OLD_EMAIL,CORRECT_NAME,CORRECT_EMAIL

  • 按Enter键运行脚本

  • 查看更改后的Git历史有没有错误

  • 把正确的历史 Push 到 Github

  • 删掉刚才临时Clone的仓库

    cd ..
rm -rf repo.git

结语

按以上步骤执行完之后，我煮熟的Contributions终于回来了，圆满解决困扰~~
